# ROLLBACK 0171–0172 (F4.4 E3/E4)

> Ejecutar SOLO en ventana autorizada por Dirección, en el SQL Editor de prod
> (`arsksytgdnzukbmfgkju`). Ambas migraciones son aditivas: el rollback es un
> drop limpio, sin enums (los enum values serían irreversibles — no se crearon).
> El código servido tolera la ausencia de las tablas (webhook responde 200 con
> `persisted:false`; el processor drena como `skipped`), así que el rollback de
> DB puede hacerse con o sin rollback de deploy.
