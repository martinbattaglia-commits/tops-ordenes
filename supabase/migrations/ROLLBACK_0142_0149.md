# ROLLBACK 0142–0149 — Nexus Link RC1.0 (fundación)

> **Naturaleza:** todas las migraciones de RC1.0 son **aditivas**. El rollback preferido es **lógico** (no destructivo). El rollback **duro** (DROP) solo si es imprescindible, en **orden inverso**, y lo ejecuta Dirección a mano (G3). DROP de buckets borra binarios (R4).
> **Proyecto:** `tops-ordenes-prod` / `arsksytgdnzukbmfgkju` (STOP ante cualquier otro).

---

## A. Rollback lógico (recomendado — sin pérdida de datos)

1. **Desactivar el módulo (gating RBAC):** el módulo queda invisible y los RPC niegan al instante.
   ```sql
   delete from public.role_permissions rp
   using public.permissions p
   where rp.permission_id = p.id and p.module = 'connect';
   -- (alternativa más suave: dejar permisos, quitar solo a roles concretos)
   ```
2. **Apagar la emisión a Knowledge** (deja de proyectar, sin tocar nada más):
   ```sql
   update public.knowledge_sources set enabled = false
   where source_table = 'connect_conversation_links';
   ```
3. **Re-deploy del código previo** (Netlify CLI). Las tablas quedan inertes sin permisos.

---

## B. Rollback duro (DROP en orden inverso — solo si imprescindible)

> Ejecutar de arriba hacia abajo. Cada bloque es independiente y defensivo (`if exists`).

```sql

drop function if exists public.project_connect_links();
drop function if exists public.knowledge_backfill_connect_links(int);
drop function if exists public.knowledge_connect_links_to_canonical(public.connect_conversation_links);
delete from public.knowledge_sources where source_table = 'connect_conversation_links';
-- (los knowledge_events ya emitidos son append-only: NO se borran salvo decisión explícita)

-- 0148 — storage (⚠️ DROP de buckets borra binarios)

drop policy if exists "connect-files write internal"  on storage.objects;
drop policy if exists "connect-files update internal" on storage.objects;
drop policy if exists "connect-files delete admin"    on storage.objects;
delete from storage.buckets where id in ('connect-files','connect-files-pii');  -- ⚠️ borra binarios

-- 0147 — notifications ext (columnas A4 + realtime)
do $$ declare t text; begin
  foreach t in array array['connect_conversations','connect_participants','connect_messages',
    'connect_message_reactions','connect_message_mentions','connect_attachments',
    'connect_conversation_links','connect_pinned'] loop
    begin execute format('alter publication supabase_realtime drop table public.%I', t);
```
