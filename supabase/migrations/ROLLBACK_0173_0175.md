# ROLLBACK ventana 0173–0175 (F5.2-lite · Nexus AI Copilot read-only)

> Ejecutar A MANO en el SQL Editor de Supabase (G3), solo con OK de Dirección.
> La ventana es 100% aditiva: ninguna tabla/función previa fue modificada, por
> lo que este rollback no afecta datos de negocio. Las tablas `ai_*` solo
> contienen auditoría propia del Copilot.

## Nivel 0 — apagado funcional (sin tocar DB, siempre disponible)

`AI_ENABLED` sin setear o distinto de `1` → el Copilot no aparece y toda
consulta corta (fail-closed). Este es el rollback de primera respuesta.

## Nivel 1 — revertir 0175 (seed)

```sql
delete from public.ai_pilot_users;
delete from public.role_permissions rp
 using public.permissions p
 where rp.permission_id = p.id and p.module = 'ai';  -- defensivo (0175 no seedea roles)
delete from public.permissions where module = 'ai';
notify pgrst, 'reload schema';
```

## Nivel 3 — 0173 (enum)

`alter type ... add value` **no es reversible** en Postgres (no existe
`drop value`). El valor `'ai'` queda en `permission_module_t` de forma inocua
(sin permisos que lo usen tras el Nivel 1). Mismo criterio aceptado en 0142
(`connect`) y 0167. No requiere acción.

## Orden recomendado

Nivel 0 (inmediato) → Nivel 1 → Nivel 2. Verificar después:
`select count(*) from pg_proc where proname like 'ai\_%';` → 0 de las listadas,
y que `/api/version` + smoke de módulos existentes siguen sanos (la ventana no
los toca, pero G5: evidencia real).
